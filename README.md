# راهنمای استفاده از سامانه ارزیابی رزومه‌ها جهت استخدام

این پروژه یک وب‌اپ تک‌فایلی/آفلاین برای غربالگری رزومه‌ها و مقایسه با شرح شغل است.

## اجرای سامانه

### اجرای محلی (بدون نصب ابزار)
1. فایل `index.html` را در مرورگر باز کنید.
2. یا فایل `CVJDHR.html` را مستقیم باز کنید.

### اجرای آنلاین روی GitHub Pages
1. به مسیر **Settings > Pages** در ریپازیتوری بروید.
2. بخش **Source** را روی branch اصلی پروژه و پوشه `/ (root)` تنظیم کنید.
3. بعد از چند دقیقه، لینک سایت فعال می‌شود.
4. برای مسیر ریشه، وجود `index.html` الزامی است (در این پروژه وجود دارد).

---

## راهنمای گام‌به‌گام کار با سامانه

### 1) ورود اطلاعات اولیه
در فرم اصلی، موارد زیر را تکمیل کنید:
- نام شرکت / سازمان
- نوع کسب‌وکار / صنعت
- عنوان شغل
- سطح سازمانی
- مدیر پاسخگو
- موقعیت جغرافیایی
- شرح شغل
- معیارهای کلیدی شایستگی

> پیشنهاد: معیارها را با ویرگول جدا کنید (مثلاً: مذاکره، مدیریت تیم، CRM، تحلیل بازار).

### 2) ورود رزومه‌ها
سه روش دارید:
- **ورود دستی متن رزومه** و ثبت نام داوطلب
- انتخاب فایل رزومه
- دکمه **ترانسکرایب فایل‌های انتخاب‌شده**: با کلیک، متن استخراج‌شده در همان کادر «متن رزومه» نمایش داده می‌شود تا قابل ویرایش دستی هم باشد.

### 3) تنظیمات سامانه
از دکمه **تنظیمات سامانه** استفاده کنید:

#### تب «هوش مصنوعی»
- انتخاب حالت پردازش داخلی یا حالت مولد
- وارد کردن API Key
- نمایش/مخفی کردن کلید
- تست اتصال
- مشاهده نتیجه تست API
- حذف کلید

#### تب «پردازش رزومه»
- انتخاب سطح تحلیل: ساده، متوسط، پیشرفته
- انتخاب زبان تحلیل: فارسی، انگلیسی، خودکار

#### تب «ظاهر سامانه»
- تغییر رنگ اصلی سامانه
- تغییر حالت نمایش: روز، شب، خودکار

### 4) اجرای تحلیل
روی دکمه **شروع تحلیل و تولید داشبورد** بزنید.

سامانه خروجی‌های زیر را نشان می‌دهد:
- KPIها (تعداد رزومه، میانگین انطباق، وضعیت پردازش)
- مشخصات شغل و معیارها
- جدول رتبه‌بندی داوطلبان
- امتیاز نهایی هر رزومه از 100
- سطح انطباق (بالا/متوسط/پایین)
- ریسک استخدام
- اولویت مصاحبه
- نقاط قوت و ضعف
- سوالات پیشنهادی مصاحبه
- پیشنهاد نهایی استخدام

### 5) خروجی گرفتن
- **خروجی متنی**: دریافت فایل `HR-Report.txt`
- **چاپ/PDF**: از طریق گزینه چاپ مرورگر

## نکات مهم
- اگر اطلاعات عنوان شغل یا رزومه ناقص باشد، سامانه کمبودها را نمایش می‌دهد و با داده موجود تحلیل می‌کند.
- امتیازدهی باید به‌عنوان کمک‌تصمیم در نظر گرفته شود، نه تصمیم نهایی قطعی.
- برای کاهش خطای استخدام، نتیجه سامانه را همراه با مصاحبه و بررسی مرجع استفاده کنید.

## فعال‌سازی امکان ساخت مجدد PR (Create PR)
اگر در GitHub دکمه **Create pull request** را نمی‌بینید یا غیرفعال است، معمولاً یکی از این حالت‌ها رخ داده است:
1. هیچ تغییر جدیدی نسبت به branch مقصد ندارید.
2. branch مبدا و مقصد یکسان انتخاب شده‌اند.
3. تغییرات هنوز commit/push نشده‌اند.

### چک‌لیست سریع
1. یک branch جدید بسازید (مثلاً: `feature/recreate-pr`).
2. حداقل یک تغییر جدید ثبت کنید (حتی اصلاح جزئی مستندات).
3. `commit` بزنید.
4. branch را `push` کنید.
5. در GitHub روی **Compare & pull request** بزنید.

### دستورات پیشنهادی
```bash
git checkout -b feature/recreate-pr
git add .
git commit -m "Enable recreate PR flow"
git push -u origin feature/recreate-pr
```

> نکته: اگر قبلاً PR بسته/مرج شده، برای ساخت PR جدید باید حتماً commit جدید داشته باشید.


## رفع مشکل نمایش خروجیِ اشتباه (نمایش `diff --git` یا علامت `+` در صفحه)
اگر صفحه‌ای که باز می‌کنید به‌جای UI سامانه، متن‌هایی مثل `diff --git`، `@@` یا علامت `+` نشان می‌دهد، معمولاً یکی از موارد زیر است:
- به‌جای لینک GitHub Pages، فایل Patch/Diff یا صفحه Compare را باز کرده‌اید.
- محتوای Patch به‌اشتباه داخل فایل HTML paste شده است.
- کش مرورگر نسخه قدیمی را نشان می‌دهد.

### مسیر درست باز کردن پروژه
1. آدرس GitHub Pages را باز کنید (نه صفحه Code/Compare/Diff).
2. برای این مخزن، فایل ورودی باید `index.html` در root باشد.
3. در صورت خطا، یک بار `Ctrl+F5` (Hard Refresh) بزنید.

### بررسی سلامت فایل‌ها قبل از Push
```bash
head -n 5 index.html
head -n 5 CVJDHR.html
```
خروجی باید با `<!doctype html>` شروع شود. اگر با `diff --git` یا `+` شروع شد، فایل خراب است و باید جایگزین شود.

### نکته تکمیلی
- فایل `404.html` در پروژه اضافه شده تا اگر مسیر اشتباه باز شد، کاربر به `index.html` هدایت شود.

## ایجاد مجدد PR به‌صورت خودکار
برای اینکه همیشه بتوانید **Create PR** را دوباره فعال کنید، اسکریپت زیر اضافه شده است:

```bash
./recreate-pr.sh feature/recreate-pr-YYYYMMDD main
```

> اگر آرگومان دوم را ندهید، اسکریپت از branch فعلی به‌عنوان base استفاده می‌کند.

این اسکریپت به‌ترتیب:
1. آخرین وضعیت remote را fetch می‌کند.
2. روی branch مبنا checkout می‌کند.
3. branch مبنا را با `pull --ff-only` به‌روز می‌کند.
4. یک branch جدید می‌سازد.
5. یک فایل marker جدید ایجاد می‌کند تا حتماً diff جدید داشته باشید.
6. commit می‌زند.
7. branch را push می‌کند و PR را قابل ایجاد می‌کند.

### روش سریع‌تر: اسکریپت اختصاصی Reopen PR
اگر فقط می‌خواهید خیلی سریع امکان ایجاد PR جدید را فعال کنید:

```bash
./reopen-pr.sh <new-branch-name> [base-branch]
```

نمونه:

```bash
./reopen-pr.sh feature/reopen-pr-20260527 main
```

این اسکریپت:
1. آخرین تغییرات `origin` را می‌گیرد.
2. روی `base-branch` می‌رود و آن را `pull --ff-only` می‌کند.
3. یک branch جدید می‌سازد.
4. یک marker تضمینی برای diff جدید ثبت می‌کند.
5. commit و push می‌کند تا دکمه **Compare & pull request** فعال شود.

### روش خیلی سریع (میانبر)
برای فعال‌سازی فوری امکان ایجاد مجدد PR می‌توانید از میانبر زیر استفاده کنید:

```bash
./enable-recreate-pr.sh <new-branch-name> [base-branch]
```

این اسکریپت یک wrapper روی `reopen-pr.sh` است و همان فرایند را اجرا می‌کند.

## آپدیت کدها + ایجاد PR جدید (خودکار)
برای سناریویی که می‌خواهید یک PR جدید با تغییر تضمینی ساخته شود، از اسکریپت زیر استفاده کنید:

```bash
./update-and-create-pr.sh <new-branch-name> [base-branch] [commit-message]
```

نمونه:

```bash
./update-and-create-pr.sh feature/new-pr-flow main "chore: update code and open new PR"
```

این اسکریپت:
1. `origin` را fetch می‌کند.
2. روی branch مبنا checkout می‌کند.
3. branch مبنا را `pull --ff-only` می‌کند.
4. branch جدید می‌سازد.
5. یک marker جدید ثبت می‌کند تا حتماً diff برای PR وجود داشته باشد.
6. commit می‌زند.
7. branch را push می‌کند.
8. لینک ساخت PR را در GitHub قابل استفاده می‌کند.

## ایجاد/به‌هنگام‌سازی فایل PR (متن آماده PR)
اگر می‌خواهید یک فایل استاندارد برای توضیحات Pull Request بسازید یا آپدیت کنید:

```bash
./create-or-update-pr-file.sh [PR_FILE] [TITLE] [SCOPE]
```

نمونه:

```bash
./create-or-update-pr-file.sh PR_MESSAGE.md "feat: improve resume analysis" "HR Dashboard"
```

خروجی این اسکریپت:
1. یک فایل Markdown برای PR می‌سازد یا همان فایل را به‌روزرسانی می‌کند.
2. بخش‌های استاندارد `Motivation`، `Description` و `Testing` را قرار می‌دهد.
3. زمان به‌روزرسانی (UTC) و Scope را در فایل ثبت می‌کند.
